Option 1

  • With reference to the Pattern 4, Vibration accelerometer sensors can connect to the off the shelf vibration monitors like IFM VSE 151 over physical wiring
  • Vibration monitor provides the following
    • Monitoring vibration values in real time with a very fast reaction time up to 1 ms and calculating dynamic values within the time range (e.g. v-RMS to ISO).
    • Calculating dynamic values within the frequency range FFT or HFFT (e.g. imbalance or rolling element bearing)
    • Process values (analogue signals) for current value above or below the limit
  • Vibration monitor provides the data over the TCP IP protocol.
  • The IIoT Gateway connects with the Vibration monitor and gets the vibration-processed data.
  • All the processed value, like FFT, Alert details are sent to the enterprise data lake and to public cloud for advanced analytics and visualization purpose.
  • By web service, the enterprise data lake communicates with public cloud.
  • For the effective analytics public cloud needs the other environmental data, like machine temperature, voltage, running condition etc., those can be extracted from the enterprise data lake to the public cloud.

Option 2

  • All the Vibration sensor data will be sent to the Vibration analyzer software such as Insight CM.
  • All the raw data processed in the software analyzer (Like Insight CM) centrally and the time domain, frequency domain values data stored in the database buffer.
  • Through the OSI PI connector, all the raw and processed value sent to Enterprise OSI PI.
  • From the Enterprise OSI PI all the processed value, like FFT, Alert details are sent to the Public Cloud for advanced analytics and visualization purpose through the web services.
  • For the effective analytics Platform needs the other environmental data, like machine temperature, voltage, running condition etc., those can be extracted from the enterprise PI connection to the Public Cloud Platform.
