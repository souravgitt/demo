The Lilly applied Architecture is based upon the following key requirements (i.e. architectural goals).

1.1.1. Functional Requirements

The applied architecture should address all the following key functional and non-functional requirements

  • Integrate different types / categories of devices and gateways that can exist considering Lilly requirements.
  • System should be capable to handle high-speed data (sampling rate <=100 ms) and low speed data (sampling rate >100 ms).
  • System should be capable to do edge analytics if require in future.
  • Data movement considerations as per criticality (red/amber/green) and GMP/Non GMP
  • System is capable to ingest data from the different external and internal data applications e.g. external cloud to internal data center
  • System is capable to ingest data in between internal data applications e.g. L3 PI to L4 enterprise PI
  • Data security requirements will be met as per Lilly security consideration

System should be capable to do device management for IIoT devices
