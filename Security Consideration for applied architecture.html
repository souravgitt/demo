Data Security, privacy and protection

  • Database security and Compliance - Data encryption at rest is a mandatory step toward data privacy and compliance. Data is managing in Lilly at the enterprise data lake level; hence follow the following best practices for data at rest.
    • Database user Management
      • Allows granular control with a username/password
      • Full Fledged Password Policy
    • Database Encryption with commercial certification like NIST to address "data in rest" directives
      • Cryptographic operations using dedicated Hardware Security Module (HSM). HSM (Hardware Security Module) are dedicated systems that physically and logically secure the cryptographic keys and cryptographic processing that are at the heart of any digital signature and data protection solution.
    • Table, Column and Row Level Access Control
    • No Data Redaction Capabilities
    • Audit-
      • Use Inbuilt Auditing (Server, Database Level and Audit Level)

Enforce Secure Network Architecture

  • A secure network to be define that denies traffic by default and allows by exception.
  • All communication happens over SSH (Secured Shell) Tunnels.
    • Inbuilt Protection against top 10 web attacks - Injection, XSS, Broken Authentication, Broken Session Management, Insecure Direct Object Reference, XSRF, Security Misconfiguration, Insecure Cryptographic Storage, Failure to restrict URL Access, Insufficient Transport Layer Protection, Invalidated Redirects and Forwards

Auditing and Logging will support logging of the following

  • Security Logging
  • Login, Logout, Failed Logins, Password Changes
  • Log Information shall contain:
    • User Id, Time Stamp and IP mandatorily.
    • Request Id and Session Id when available
    • Log changes to password policy.
    • Entity level Logging (Created By, Create Time, Edited By, Edit Time)
    • File based Logs for code (ERROR/INFO/WARN/DEBUG levels)
